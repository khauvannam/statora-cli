- [ ] `statora switch --dry-run` (print plan without executing)

### Root flag
- [ ] `--version` / `-v` flag on root (print statora version string)

### Dispatch: composer should run under PHP
- [ ] `dispatch.Dispatch("composer")` should exec `phpBin composerPhar ...args` not just `composerPhar ...args` (spec §9: `syscall.Exec(phpBin, [phpBin, composerPhar, ...], env)`)

### Structural gaps vs spec
- [ ] `internal/dispatch/rescache/` — spec shows as subdirectory (`rescache/cache.go`, `rescache/invalidate.go`); we have `internal/dispatch/rescache.go` as flat file — functional but doesn't match spec layout
- [ ] `SwitchPlan` struct shape — spec defines `PHPChange *VersionChange`, `ComposerChange *VersionChange`, `ExtActions []ExtAction`; we use a flat `[]Action` slice — works but diverges from spec shape
- [ ] Extension `InvalidateCacheStage` — not a pipeline stage; called directly in `switcher.Execute` — minor gap
